# 第一阶段 · 机器学习基线方法

将每天的气象观测视为**独立样本**，通过多变量特征工程训练经典机器学习模型来预测日均气温。

## 使用模型
- 决策树（Decision Tree）
- 随机森林（Random Forest）
- **XGBoost**（最优）

## 建模思路
- 输入特征：最高气温、最低气温、降水量、风速、辐射量等多变量
- 建模前进行系统的探索性数据分析（EDA）和特征工程
- 使用交叉验证评估模型泛化能力

## 实验结果

| 模型 | R² | RMSE (°C) | MAE (°C) |
|------|----|-----------|----------|
| 多元线性回归 | 0.898 | 2.76 | 2.19 |
| 决策树 | 0.877 | 3.04 | 2.44 |
| XGBoost | **0.894** | **2.82** | **2.28** |

## 方法局限性

这一阶段的核心局限在于**忽略了时间顺序**——每一天被当作独立样本处理，模型无法感知气温序列内在的自相关结构和季节性规律。换言之，即便打乱数据的日期顺序，模型的预测结果也不会有任何变化。

这正是引入第二阶段时间序列与深度学习方法的动因——详见 `part2_time_series_deep_learning/`。
